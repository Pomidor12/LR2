# Отчет по лабораторной работе №2
Цель данной лабораторной работы — освоить практические навыки автоматизации процесса непрерывной интеграции и развертывания (CI/CD) с использованием Jenkins — мощного инструмента автоматизации сборки, а также наладить взаимодействие с удалённым репозиторием GitHub посредством webhook, обеспечивающим мгновенный запуск сборки при каждом изменении кода.
## Ход работы:
1. Авторизация на Jenkins-сервере с выданными данными от преподователя
2. Создали новый проект и задали уникальное имя проекта. Далее в списке типов проектов выбрали Pipeline и нажали OK.
3. В открывшемся окне переходим к разделу Pipeline:
   3.1. В поле Definition выбераем Pipeline script. В поле ниже вставляем заготовленный Pipeline скрипт (Jenkinsfile), предоставленный преподавателем.
   3.2. В Jenkinsfile отдельно редактируем параметры: В поле STUDENT_NAME указываем своё имя, в поле PORT укажите уникальный порт.
   <img width="1003" height="328" alt="image" src="https://github.com/user-attachments/assets/9529d32f-72fa-449c-bff9-f85bfc93d628" />
4. Далее настраиваем GitHub webhook для автоматического запуска сборки.
<img width="782" height="151" alt="image" src="https://github.com/user-attachments/assets/7ecaf67a-788a-49e1-b872-c341e79de285" />
5. В интерфейсе Jenkins открываем созданную job. Нажмаем кнопку Build Now, чтобы инициировать первую сборку. Мониторим процесс исполнения через вкладку Console Output — убеждаемся, что код корректно выкачан, прошли тесты, приложение собрано и запущено. Далее открываем приложение в браузере по адресу – http://84.201.147.67:8096/
6. Вносим изменения в репозиторий (поменяли цвет и текст) и выполнили коммит в ветку master.
7. Убеждаемяь, что GitHub webhook корректно сработал, и Jenkins автоматически запустил повторную сборку.
